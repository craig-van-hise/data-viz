
### 🤖 AGENT TASK: Refine Tooltip Viewport Collision Logic (Allow Header Overlap)

**Target File:** `composers.html`

**Objective:** Fix a bottom-clipping issue caused by overly aggressive top-boundary protection. The tooltip should default to popping UP (even if it visually overlaps the sticky `.timeline-header`) and must strictly avoid clipping outside the `window.innerHeight` or absolute `rect.top`.

**Implementation Steps:**

1. Locate the `marker.addEventListener('mouseenter', function () { ... })` block inside the `createRows()` function.
2. Replace the existing collision logic with this streamlined, absolute viewport logic:

```javascript
const rect = this.getBoundingClientRect();
const tooltip = this.querySelector('.film-tooltip');

// 1. Calculate absolute space to the edge of the browser window (ignore internal headers)
const spaceAboveWindow = rect.top; 
const spaceBelowWindow = window.innerHeight - rect.bottom;

// 2. Safe height estimate for the tooltip
const SAFE_HEIGHT = 260; 

// 3. The Math: Only flip DOWN if we are about to clip the absolute top of the browser 
// AND the bottom actually has more space than the top.
if (spaceAboveWindow < SAFE_HEIGHT && spaceBelowWindow > spaceAboveWindow) {
    tooltip.classList.add('tooltip-down');
} else {
    // Default to UP. It is acceptable if it visually overlaps the page header.
    tooltip.classList.remove('tooltip-down');
}
```

**Validation:** Ensure the `headerBottom` logic from the previous iteration is completely removed. The script should only reference `rect.top` and `window.innerHeight`.
