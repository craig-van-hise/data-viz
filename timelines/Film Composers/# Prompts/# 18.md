#### Task 1: Create the Data Audit Script
Create a new Python script at `scripts/audit_youtube_links.py`. This script will iterate through all JSON files in `data/filmographies/` and validate the `youtube_link` using `yt-dlp` to fetch metadata.

**Script Requirements:**
1.  **Dependencies:** Use `json`, `os`, `re`, and `yt_dlp`. (Assume the user will run `pip install yt-dlp`).
2.  **Target Directory:** Loop through all `.json` files in `data/filmographies/`. The composer's name can be derived from the filename (e.g., `Elliot_Goldenthal.json` -> Last name is "Goldenthal").
3.  **Data Extraction:** For each item in the JSON that has a non-null `youtube_link`, use `yt-dlp` (`extract_info` with `download=False`) to fetch the `title` and `description` of the video. Wrap this in a `try/except` block; if the video is private, deleted, or errors out, automatically treat it as a failed validation.
4.  **Normalization Logic:** * Convert the film title (from the JSON), the YouTube title, and the YouTube description to lowercase.
    * Strip special characters and punctuation from all three strings using regex (keep only alphanumeric characters and spaces).
5.  **Validation Logic:**
    * *Check A:* Does the normalized film title exist in the normalized YouTube title as a distinct word/phrase? (Use regex word boundaries `\b`). If True, keep the link.
    * *Check B:* If Check A fails, does the normalized YouTube description contain BOTH the normalized film title AND the composer's last name? If True, keep the link.
    * *Failure:* If both checks fail, update the item's dictionary: `"youtube_link": null`.
6.  **File Writing:** Save the updated JSON data back to the original file, ensuring proper indentation (`indent=4`).
