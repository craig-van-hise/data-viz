
### 🤖 AGENT TASK: Implement Airtight Tooltip Collision Priority

**Target File:** `composers.html`

**Objective:** Fix a critical clipping bug where the tooltip is forced into the top edge of the browser if both the top and bottom of the viewport are constrained. Implement a strict "greatest available space" priority algorithm.

**Implementation Steps:**

1. Locate the `marker.addEventListener('mouseenter', function () { ... })` block inside the `createRows()` function.
2. Replace the entire contents of that event listener with this foolproof logic:

```javascript
const rect = this.getBoundingClientRect();
const tooltip = this.querySelector('.film-tooltip');

const spaceAbove = rect.top;
const spaceBelow = window.innerHeight - rect.bottom;

// A generous maximum height for the tooltip (Image + Text + Padding + Offset)
const TOOLTIP_MAX_HEIGHT = 300; 

let flipDown = false;

// If we don't have enough room to safely pop UP...
if (spaceAbove < TOOLTIP_MAX_HEIGHT) {
    // ...check if we have enough room to go DOWN, OR if going down is just the lesser of two evils.
    if (spaceBelow >= TOOLTIP_MAX_HEIGHT || spaceBelow > spaceAbove) {
        flipDown = true;
    }
}

if (flipDown) {
    tooltip.classList.add('tooltip-down');
} else {
    tooltip.classList.remove('tooltip-down');
}
```

**Validation:** Replace the previous bounding client rect logic entirely. This guarantees that if a clip is inevitable due to a tiny viewport, it will *always* default to the side that reveals the most of the poster.
