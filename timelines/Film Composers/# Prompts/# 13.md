
# Agent Task: CSS Stacking Context Z-Index Fix

**Context:** The SVG poles for the "Most Important Films" are rendering in front of the labels of lower lanes. The previous z-index fix failed because the descending z-index was applied to the child `.marker-imp-container`, trapping it inside the parent `.film-marker`'s stacking context. We need to apply the calculated z-index to the parent `.film-marker` element to correctly layer the elements globally.

**File to Modify:** `composer_profile.html`

**Instructions:**

### 1. Elevate Z-Index to the Parent Marker
Locate the `resolveSmartCollisions` function. Find Step 4 (where the container is positioned and the z-index is currently set).

**Find this line:**
```javascript
                box.style.zIndex = 1000 - (laneIndex * 10);
```

**Replace it with these lines:**
```javascript
                // Reset the container's z-index so it doesn't create a conflicting context
                box.style.zIndex = 'auto';
                
                // Apply the descending z-index directly to the parent .film-marker
                // Higher lanes get a lower z-index so their poles drop behind lower lanes.
                marker.style.zIndex = 1000 - (laneIndex * 10);
```
