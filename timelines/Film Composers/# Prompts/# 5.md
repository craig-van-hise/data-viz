
# Agent Task: Implement Dynamic Vertical Stacking for Timeline Markers

**Context:** The current timeline visualization in `composer_profile.html` limits film markers to a 3-row staggered layout using a modulo operator (`% 3`). This causes visual data loss for prolific composers like Max Steiner, who often had more than three scores in a single year. We need to transition to a dynamic column-grouping approach that stacks dots vertically from the center of the track, regardless of the number of entries.

**File to Modify:** `composer_profile.html`

**Instructions:**

1.  **Locate the `drawMarkers` function.** 2.  **Find the block labeled `// Apply Staggering`** (currently starts around line 378).
3.  **Replace the existing staggering logic** with the following dynamic grouping and centering algorithm:

```javascript
            // Apply Staggering (Dynamic Vertical Stacking)
            markers.sort((a, b) => a.pct - b.pct);

            // 1. Group markers into columns if they are close together horizontally
            const columns = [];
            let currentColumn = [];

            markers.forEach(m => {
                if (currentColumn.length === 0) {
                    currentColumn.push(m);
                } else {
                    const head = currentColumn[0];
                    // If within 1.5% of the column's start, add to the vertical stack
                    if (m.pct - head.pct < 1.5) { 
                        currentColumn.push(m);
                    } else {
                        columns.push(currentColumn);
                        currentColumn = [m];
                    }
                }
            });
            if (currentColumn.length > 0) columns.push(currentColumn);

            // 2. Distribute each column vertically from the center
            columns.forEach(col => {
                const count = col.length;
                const spacing = 16; // Vertical pixels between dots
                
                // Calculate starting offset to keep the stack centered on the track
                const startOffset = -((count - 1) * spacing) / 2;
                
                // Use the column's average percentage for perfectly straight vertical lines
                const avgPct = col.reduce((sum, m) => sum + m.pct, 0) / count;

                col.forEach((m, j) => {
                    m.el.style.left = `${avgPct}%`; 
                    const offset = startOffset + (j * spacing);
                    
                    // Center the stack on the 50% line of the track
                    m.el.style.top = `calc(50% + ${offset}px)`;
                    track.appendChild(m.el);
                });
            });
```

**Technical Requirements for the Agent:**
* **Variable Consistency:** Ensure the script maintains the use of the `markers`, `track`, and `m.el` variables already established in the scope.
* **Alignment:** The `avgPct` calculation is critical to ensure that even if films have slightly different release dates within the same month, they appear in a single, clean vertical line.
* **Overflow Handling:** Note that dots may now extend vertically beyond the `60px` track container for very busy years; this is intended behavior to ensure all data is visible.
